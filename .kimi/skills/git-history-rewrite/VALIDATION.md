# 验收清单

## 验证环境准备
- [ ] 确认有多次提交历史（至少3次）
- [ ] 确认可以修改提交历史

## 自然语言触发测试

| 触发词 | 预期行为 | 通过 |
|--------|----------|------|
| "修正提交信息" | 提供修改提交信息的方法 | ☐ |
| "修改历史提交" | 指导如何修改历史 | ☐ |
| "提交信息改中文" | 提供中英文转换方案 | ☐ |
| "amend commit" | 说明 amend 用法 | ☐ |

## 功能验收清单

### 修改最近一次提交
- [ ] 提供 `git commit --amend` 命令
- [ ] 说明如何修改提交信息

### 修改历史提交
- [ ] 提供 `git log` 查看提交 hash
- [ ] 提供 `git checkout <commit>` 切换到目标提交
- [ ] 提供 `git commit --amend` 修改该提交
- [ ] 提供 `git cherry-pick` 后续提交
- [ ] 提供 `git branch -f` 重置分支指针

### 安全提醒
- [ ] 提醒创建备份分支
- [ ] 提醒强制推送风险
- [ ] 说明需要逐个处理

### 禁用方法说明
- [ ] 说明为什么不推荐 `git rebase -i`
- [ ] 说明为什么不推荐 `git filter-branch`

## 评分标准

- 全中（100%）：所有检查项通过
- 良好（80%）：核心功能正常，缺少部分安全提醒
- 及格（60%）：基本能指导修改，步骤不够详细
- 失败（<60%）：提供的方法有误或危险
