# 待办事项管理助手

智能管理任务列表，支持优先级、状态追踪和进度统计。

## 触发条件

当用户提及以下关键词时触发：
- "管理待办"、"管理任务"
- "创建任务"、"添加任务"、"新建待办"
- "查看todo"、"列出任务"、"显示待办"
- "更新任务"、"修改任务"
- "完成任务"、"标记完成"
- "跟踪进度"、"任务统计"

## 执行流程

### 1. 识别用户意图

| 用户意图 | 关键词 | 操作 |
|----------|--------|------|
| 创建任务 | "创建"、"添加"、"新建" | 创建新任务 |
| 查看任务 | "查看"、"列出"、"显示" | 展示任务列表 |
| 更新任务 | "更新"、"修改"、"编辑" | 修改任务信息 |
| 完成任务 | "完成"、"标记"、"删除" | 更新任务状态 |
| 统计进度 | "统计"、"进度"、"报告" | 生成进度报告 |

### 2. 任务属性

每个任务包含：
- **标题**：任务名称（必需）
- **描述**：详细说明（可选）
- **优先级**：高/中/低，默认中
- **状态**：待办/进行中/已完成
- **截止日期**：可选
- **标签**：用于分类，可选

### 3. 存储方式

- 优先使用项目根目录的 `TODO.md`
- 若无则创建新文件
- 格式：Markdown 任务列表

### 4. 响应格式

- 简洁确认操作结果
- 列表展示时带编号和状态图标
- 统计时显示完成率百分比

## 示例

### 创建任务
```
用户：创建任务"修复登录bug"，优先级高
→ 在TODO.md中添加：
   - [ ] 🔴 修复登录bug
```

### 查看任务
```
用户：查看所有待办
→ 显示：
   1. ☐ 🔴 修复登录bug
   2. ☑ 🟡 更新文档
   3. ☐ 🟢 优化代码
```

### 完成任务
```
用户：标记第一个任务完成
→ 更新为：- [x] 🔴 修复登录bug
```

### 进度统计
```
用户：查看任务进度
→ 显示：总计5个，已完成3个，完成率60%
```
