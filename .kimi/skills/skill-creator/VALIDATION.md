# 验收清单

## 验证环境准备
- [ ] 确认可以创建新目录和文件
- [ ] 准备测试用的 skill 名称（如 test-skill）

## 自然语言触发测试

| 触发词 | 预期行为 | 通过 |
|--------|----------|------|
| "创建 skill" | 指导创建 skill 的完整流程 | ☐ |
| "新建技能" | 创建技能目录和文件 | ☐ |
| "写 skill" | 提供 skill 编写指导 | ☐ |
| "添加技能" | 创建新技能结构 | ☐ |

## 功能验收清单

### 目录结构创建
- [ ] 创建 `skill-name/` 目录
- [ ] 创建 `SKILL.md` 文件
- [ ] **创建 `VALIDATION.md` 文件（硬性要求）**

### SKILL.md 内容
- [ ] 包含 YAML frontmatter（name + description）
- [ ] description 包含触发词
- [ ] 有触发条件说明
- [ ] 有执行流程
- [ ] 有示例

### VALIDATION.md 内容
- [ ] 有验证环境准备
- [ ] 有自然语言触发测试
- [ ] 有功能验收清单
- [ ] 有评分标准

### 指导原则
- [ ] 强调简洁（<5k 词）
- [ ] 强调触发词设计
- [ ] 提供检查清单

## 评分标准

- 全中（100%）：所有检查项通过，包含 VALIDATION.md
- 良好（80%）：创建了基本结构，VALIDATION.md 不够详细
- 及格（60%）：创建了 SKILL.md，缺少 VALIDATION.md
- 失败（<60%）：结构不完整或不符合规范

## 硬性要求

**必须创建 VALIDATION.md，否则视为失败！**
