# 代码质量与重构助手

帮助识别代码异味，应用设计原则进行安全重构。

## 核心原则

### DRY (Don't Repeat Yourself)
> 系统中的每一处知识都必须拥有单一、明确、权威的表示。

### SRP (Single Responsibility Principle)
> 一个函数/类只做一件事，只有一个变化理由。

### OCP (Open/Closed Principle)
> 对扩展开放，对修改关闭。

## 重构流程

### Step 1: 识别代码异味

检查以下信号：

| 异味 | 症状 | 检查方法 |
|------|------|----------|
| **重复代码** | 相同/相似代码多处出现 | 修改一个逻辑需要改几处？ |
| **过长函数** | 函数超过 50 行 | 能否用一句话描述函数功能？ |
| **魔法数字** | 硬编码数字/字符串 | 数字是否有明确含义？ |
| **过大类** | 类做太多事 | 类名是否准确反映所有职责？ |
| **发散式变化** | 修改需求要改多个地方 | 加新功能要改旧代码吗？ |

### Step 2: 选择重构手法

| 问题 | 手法 | 示例 |
|------|------|------|
| 重复代码 | 提取函数、上移方法 | 将重复逻辑提取为公共函数 |
| 过长函数 | 提取函数、替换算法 | 按步骤拆分为小函数 |
| 魔法数字 | 提取常量、配置化 | `const MAX_SIZE = 100` |
| 过大类 | 提取类、分离职责 | 按职责拆分为多个类 |
| 条件分支多 | 多态、策略模式 | 用策略对象替代 switch |

### Step 3: 执行重构

**Before & After 示例：**

```javascript
// ❌ 重复代码 + 魔法数字
function calculateOrderDiscount(orderTotal) {
  if (orderTotal > 100) return orderTotal * 0.1;
  return 0;
}
function calculateCouponDiscount(couponTotal) {
  if (couponTotal > 100) return couponTotal * 0.1;
  return 0;
}

// ✅ 提取函数 + 常量
const DISCOUNT_THRESHOLD = 100;
const DISCOUNT_RATE = 0.1;

function calculateDiscount(amount) {
  return amount > DISCOUNT_THRESHOLD ? amount * DISCOUNT_RATE : 0;
}
```

### Step 4: 验证

- [ ] 重构前后行为一致
- [ ] 单元测试通过
- [ ] 代码更易读
- [ ] 没有过度抽象

## 常见陷阱

| 陷阱 | 症状 | 解决 |
|------|------|------|
| **过早抽象** | 只有一次使用就抽象 | 等到有明确重复时再抽象 |
| **过度抽象** | 参数过多、难以理解的通用函数 | 保持简单，参数 ≤ 4 个 |
| **不完全替换** | 部分使用新抽象，部分保留旧代码 | 一次性替换所有重复点 |

## 快速检查清单

重构前问自己：
- [ ] 这段代码修改时需要改几处？
- [ ] 这个函数能做一件事吗？
- [ ] 这些数字/字符串有明确含义吗？
- [ ] 加新功能要改旧代码吗？

**2项以上勾选 → 考虑重构**

## 最佳实践

1. **小步快跑** - 每次只改一点，频繁提交
2. **先写测试再重构** - 确保不破坏功能
3. **命名即文档** - `calculateDiscount` 比 `calc` 好
4. **渐进式重构** - 不要一次性重构整个代码库
