# 验收清单

## 验证环境准备
- [ ] 准备有代码异味的示例代码（重复代码、过长函数、魔法数字等）
- [ ] 确认可以修改测试文件

## 自然语言触发测试

| 触发词 | 预期行为 | 通过 |
|--------|----------|------|
| "重构" | 分析代码并提供重构建议 | ☐ |
| "代码质量" | 检查代码质量问题 | ☐ |
| "消除重复" | 识别重复代码并建议提取 | ☐ |
| "DRY" | 应用 DRY 原则优化代码 | ☐ |
| "优化代码" | 提供代码优化方案 | ☐ |

## 功能验收清单

### 代码异味识别
- [ ] 能识别重复代码（copy-paste）
- [ ] 能识别过长函数（>50行）
- [ ] 能识别魔法数字（硬编码数值）
- [ ] 能识别过大类（职责过多）

### 重构建议
- [ ] 提供提取函数建议
- [ ] 提供提取常量建议
- [ ] 提供拆分函数建议
- [ ] 提供配置化建议

### 输出格式
- [ ] 指出具体问题位置（文件名:行号）
- [ ] 给出重构前代码示例
- [ ] 给出重构后代码示例
- [ ] 说明重构理由

### 设计原则
- [ ] 提及 DRY 原则
- [ ] 提及 SRP（单一职责）
- [ ] 提及 OCP（开闭原则）

## 评分标准

- 全中（100%）：所有检查项通过
- 良好（80%）：核心功能正常，部分建议不够详细
- 及格（60%）：基本能识别问题，建议不够具体
- 失败（<60%）：无法正确识别代码异味
