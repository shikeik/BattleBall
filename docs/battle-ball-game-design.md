# 球球大作战彩豆场景 - 策划案

## 项目信息
- **分支**: feature/battle-ball-game
- **目标**: 实现球球大作战核心玩法 - 吃彩豆变大
- **技术栈**: WebGPU + JavaScript

---

## 第一阶段：基础框架

### 目标
搭建 BattleBallScreen 基础框架，能进入场景并显示空白地图。

### 内容
1. 创建 `js/screens/battle-ball-screen.js`
2. 继承 Screen 类
3. 地图大小 4000×4000
4. 绘制网格线（100×100）和边界线
5. 相机跟随玩家（先放一个静态玩家球）
6. 调试面板加相机缩放滑动条

### 验收标准
- [ ] 能从主菜单进入 BattleBallScreen
- [ ] 能看到网格和边界
- [ ] 相机缩放滑动条能调节视距
- [ ] 返回键能正常返回主菜单

---

## 第二阶段：彩豆生成

### 目标
生成 80 万彩豆，使用 WebGPU 实例化渲染。

### 内容
1. WebGPU 初始化（device、pipeline、bind group）
2. 彩豆数据结构：
   - 位置 (x, y)
   - 颜色（随机 RGB）
   - 边形（3-6 随机）
   - 旋转角度（随机）
   - 半径（固定 5）
3. 空间分割：40×40 网格索引
4. 视锥剔除：只渲染屏幕内的彩豆

### 验收标准
- [ ] 进入场景能看到 80 万彩豆（视口内部分）
- [ ] 移动相机能看到不同区域的彩豆
- [ ] 帧率保持在 60fps

---

## 第三阶段：玩家控制

### 目标
实现玩家移动和吃彩豆。

### 内容
1. 虚拟摇杆 UI（左下角）
2. 玩家移动逻辑：
   - 速度随半径增大而减小
   - 边界限制
3. 碰撞检测：
   - 玩家与彩豆的距离检测
   - 使用空间分割优化
4. 吃彩豆效果：
   - 彩豆消失
   - 玩家半径增加

### 验收标准
- [ ] 虚拟摇杆能控制玩家移动
- [ ] 玩家能吃到彩豆并变大
- [ ] 玩家不能移出地图边界

## 技术细节

### 彩豆数据结构（每个 6 float = 24 bytes）
```
80万 × 24字节 = 19.2 MB GPU 内存
```

| 字段 | 类型 | 说明 |
|------|------|------|
| x | float32 | 位置 X |
| y | float32 | 位置 Y |
| color | uint32 (packed) | RGBA 颜色 |
| sides | uint32 | 边形 3-6 |
| rotation | float32 | 旋转角度 |
| radius | float32 | 半径（固定5） |

### 空间分割
- 网格大小：100×100
- 地图划分：40×40 = 1600 个格子
- 每个格子存储彩豆索引数组

### 相机缩放范围
- 最小：0.5x（看到更多地图）
- 最大：3x（聚焦玩家）
- 默认：1x

---

## 注意事项

1. **WebGPU 兼容性**：确保浏览器支持 WebGPU
2. **内存管理**：80 万彩豆占用约 20MB GPU 内存，注意低端设备
3. **性能监控**：每阶段完成后检查帧率
4. **代码质量**：遵循项目规范（Tab 缩进、logger 输出）
